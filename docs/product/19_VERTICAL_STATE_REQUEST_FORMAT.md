# Vertical State Request Format

The vertical state request is the structured bridge between a vertical pack and the future product runtime.

It includes:

- vertical identity;
- objective;
- domains;
- entities;
- evidence types;
- gates;
- cube faces;
- sample input;
- expected outputs;
- blocked actions.

This format allows the product to ask the same core question for multiple operations:

```text
What is the operational state and what is the next safe movement of the cube?
```
